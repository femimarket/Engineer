# v1 — Production Readiness

State of `ContentView` as of v1 ship. Pairs with `doc.md`.

## Shipped (blockers cleared)

- **Library has zero secrets.** `bearer` is a required init parameter.
  Demo bearer lives only in `ImageIterate2App` (the sample app), never in
  `ContentView.swift`. Production callers source from Keychain or a
  server-issued session.
- **All inputs parameterized.** `init(initialImage:, initialFilename:,
  bearer:, onCommit:)` — no defaults, caller supplies everything explicitly.
- **No log leakage in Release.** `log()` wrapped in `#if DEBUG` with
  `@autoclosure` so the message string isn't constructed in Release builds.
- **Memory ceiling.** Variations rail uses `LazyHStack` (off-screen cards
  don't decode) and caps at 30 most-recent (`variationCeiling`); oldest
  drops off when new ones generate. Worst-case ~120 MB on 1024×1024 PNGs.
- **VoiceOver labels** on icon buttons: header refresh ("Reset"), fullscreen
  X ("Close").
- **Single dismiss callback.** `.onDisappear` fires `onCommit(CommitResult)`
  on every dismiss path (parent binding, swipe-down, programmatic).
- **Swift 6.2 + iOS 26.** Strict concurrency clean (`log` / `scrub` marked
  `nonisolated` since they're pure).
- **Builds two ways:** Xcode app target and as a Swift Package library
  (`Package.swift` at project root, depends on local `Generated/Api`).
- **Result fetches go through `MediaApi`** — company-canonical media client
  with built-in URL cache (64 MB RAM / 256 MB disk), connection pooling, and
  30s timeout. Repeated fetches of the same filename hit cache.

## Known limitations (deferred, non-blocking)

- **Server response content-type not validated.** `fetchResultImage` decodes
  raw bytes via `UIImage(data:)`; if `femi.market` returns an HTML error
  page or unexpected payload, decode returns nil and throws
  `ImageServiceError.decodingFailed` with no diagnostic context.
- **No retry / backoff.** Transient API failures surface as a 3-second red
  pill banner — no automatic recovery.
- **No unit tests.**
- **No state restoration.** App backgrounded mid-session loses hero,
  history, variations.
- **In-memory only.** Nothing is written to disk by this screen for the
  iteration state (`MediaApi`'s URL cache handles fetched bytes via the
  shared cache, not screen-owned persistence). Caller owns explicit
  persistence via the `onCommit` callback.

## Public API contract

```swift
public init(
    initialImagePath: String,
    bearer: String,
    onCommit: @escaping (_ heroData: Data) -> Void
)
```

- Disk path loaded once into `Data` + `UIImage`. Basename used as the
  internal server filename for the first generation; never exposed.
- Bearer goes into `ApiAPIConfiguration.shared.customHeaders["Authorization"]`
  once on init.
- `onCommit` fires once on `.onDisappear` with the kept image's raw bytes —
  primitive only.

### Future revisit
- **Lineage**: not exposed today. Internal `historyFiles` tracks promoted
  heroes for the back-strip and revert. If a future caller needs the chain
  of choices, add a second callback param.

## Future versions

- **v1.1**: add retry/backoff; content-type guard on result fetch.
- **v2**: secret multi-keep (long-press → Save/Discard, heart badge,
  `CommitResult.savedFilenames`). See `doc.md` for full design.
- **v2+**: in-app gallery of past sessions, export-to-Photos as a separate
  downstream verb, state restoration for backgrounded sessions.
